📚 Aprende Arreglos

🗄️ ¿Qué es un arreglo?

Un arreglo es una estructura de datos que permite almacenar varios valores bajo un mismo nombre, organizados en posiciones numeradas llamadas índices.

Imagínalo como una fila de lockers numerados: cada locker guarda un dato y su número es la dirección para acceder a él.

💡 Analogía: los lockers

[1]42
[2]Hola
[3]vacío

Cada posición del arreglo puede guardar un dato diferente: un número, una palabra, un símbolo, etc.

📝 Declarar un arreglo en pseudocódigo

Para crear un arreglo indicamos su nombre y cuántos elementos va a guardar:

{ Arreglo vacío de 5 posiciones } ARREGLO lockers[5] { Arreglo con valores iniciales } ARREGLO frutas[3] ← ["manzana", "pera", "uva"] { El tamaño queda fijo al declararlo } { frutas tiene exactamente 3 posiciones }

La flecha ← significa "se le asigna el valor". Es la notación típica de pseudocódigo.

🔢 Los índices empiezan en 0

En los arreglos el conteo siempre empieza desde 0, no desde 1. El primer elemento está en la posición 0, el segundo en la 1, y así sucesivamente.

[0]🐶
[1]🐱
[2]🐭
[3]🐹
ARREGLO animales[4] ← ["Perro", "Gato", "Ratón", "Hámster"] { Leer el primer elemento (índice 0): } MOSTRAR animales[0] → "Perro" { Leer el tercer elemento (índice 2): } MOSTRAR animales[2] → "Ratón" { ¡Cuidado! Índice fuera de rango: } MOSTRAR animales[5] → ERROR (no existe)

📏 Tamaño del arreglo

El tamaño (o longitud) indica cuántas posiciones tiene el arreglo. El índice del último elemento es siempre tamaño - 1.

ARREGLO colores[3] ← ["rojo", "verde", "azul"] { El tamaño es 3 } { Los índices válidos son: 0, 1, 2 } { Acceder al último elemento: } MOSTRAR colores[3 - 1] → "azul" { En general: colores[tamaño - 1] }

✏️ Guardar un valor (escritura)

Para guardar un dato en el arreglo indicamos el nombre, el índice entre corchetes, y el valor que queremos almacenar:

ARREGLO lockers[3] { Guardar "Libro" en la posición 0: } lockers[0] ← "Libro" { Guardar 99 en la posición 2: } lockers[2] ← 99 { Estado del arreglo: } { lockers → ["Libro", vacío, 99] }

👁️ Leer un valor (lectura)

Para leer un dato usamos el nombre del arreglo y el índice de la posición que queremos consultar:

ARREGLO notas[4] ← [85, 90, 78, 95] { Leer y guardar en una variable: } resultado ← notas[1] MOSTRAR resultado → 90 { Usar directamente en una condición: } SI notas[3] > 89 ENTONCES MOSTRAR "Excelente" FIN SI

🔄 Modificar un valor existente

Asignar un nuevo valor a una posición que ya tenía datos simplemente lo reemplaza:

ARREGLO frutas[3] ← ["manzana", "pera", "uva"] { Reemplazar "pera" por "mango": } frutas[1] ← "mango" { Resultado: } { frutas → ["manzana", "mango", "uva"] }

🔄 Recorrer un arreglo

Recorrer significa visitar cada posición del arreglo una por una. Se usa una estructura DESDE … HASTA (equivalente al ciclo for):

ARREGLO frutas[3] ← ["manzana", "pera", "uva"] { Mostrar todos los elementos: } DESDE i ← 0 HASTA 2 HACER MOSTRAR frutas[i] FIN DESDE { Salida: } { "manzana" "pera" "uva" }

➕ Acumular valores

Podemos usar el recorrido para calcular la suma, el mayor, el menor, o cualquier resultado sobre todos los datos:

ARREGLO notas[5] ← [70, 85, 90, 60, 95] suma ← 0 { Sumar todas las notas: } DESDE i ← 0 HASTA 4 HACER suma ← suma + notas[i] FIN DESDE promedio ← suma / 5 MOSTRAR promedio → 80

🔍 Buscar un elemento

Para saber si un valor existe en el arreglo recorremos todas las posiciones comparando una por una:

ARREGLO nums[4] ← [10, 25, 30, 15] buscar ← 25 encontrado ← FALSO DESDE i ← 0 HASTA 3 HACER SI nums[i] = buscar ENTONCES encontrado ← VERDADERO MOSTRAR "Encontrado en posición", i FIN SI FIN DESDE SI NO encontrado ENTONCES MOSTRAR "No está en el arreglo" FIN SI
